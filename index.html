A arquitetura em camadas separa responsabilidades para manter o código organizado, testável e fácil de manter.

📂 Estrutura do Projeto

src/main/java/com.seuprojeto
├── controller
├── service
├── repository
├── entity
└── dto

Cada pacote tem uma responsabilidade clara. Essa organização facilita manutenção, testes e escalabilidade.

🔎 Responsabilidade de Cada Camada

Controller

Recebe requisições HTTP e delega ao Service.

Service

Aplica regras de negócio e coordena operações.

Repository

Realiza operações no banco de dados.

DTO & Entity

DTO transporta dados. Entity representa a tabela.

Como uma requisição percorre o sistema?

Exemplo: um usuário envia um POST /users para criar um novo cadastro.

💻 Exemplo Simplificado

@PostMapping
public ResponseEntity create(@RequestBody UserDTO dto) {
    return ResponseEntity.ok(userService.save(dto));
}

O Controller recebe o DTO e delega ao Service, que aplica regras e salva usando o Repository.

Componentes Estruturais da Aplicação

Elementos que sustentam a aplicação, mas não participam diretamente do fluxo da requisição.

Config

Configurações globais do sistema.

  • Segurança
  • CORS
  • Beans personalizados

Exception

Tratamento centralizado de erros.

  • @RestControllerAdvice
  • Padronização de respostas
  • Evita try/catch espalhado

🧠 Resumo

✔ Controller não contém regra de negócio
✔ Service concentra lógica
✔ Repository acessa o banco
✔ DTO protege dados sensíveis
✔ Separação facilita manutenção
